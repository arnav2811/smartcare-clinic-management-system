# Smart Clinic Management System - User Stories

## Overview

This document contains the user stories for the Smart Clinic Management System. The stories are organized by user role: Admin, Patient, and Doctor. These user stories define the functional requirements of the system and help guide the development process.

---

# Admin User Stories

## US-01: Manage Doctor Accounts

**As an Admin,**
I want to create, update, and remove doctor accounts,
**so that** the clinic maintains accurate doctor information.

---

## US-02: Manage Patient Accounts

**As an Admin,**
I want to view and manage patient records,
**so that** patient information remains up to date.

---

## US-03: Monitor Appointments

**As an Admin,**
I want to view all appointments,
**so that** I can monitor clinic activity and scheduling.

---

## US-04: Manage System Access

**As an Admin,**
I want to control user access and permissions,
**so that** only authorized users can access the system.

---

# Patient User Stories

## US-05: Register an Account

**As a Patient,**
I want to register for an account,
**so that** I can access clinic services online.

---

## US-06: Book an Appointment

**As a Patient,**
I want to book an appointment with a doctor,
**so that** I can receive medical consultation.

---

## US-07: View Appointment History

**As a Patient,**
I want to view my appointment history,
**so that** I can keep track of previous consultations.

---

## US-08: Cancel an Appointment

**As a Patient,**
I want to cancel an appointment,
**so that** I can manage changes to my schedule.

---

## US-09: View Prescriptions

**As a Patient,**
I want to access my prescriptions online,
**so that** I can review my treatment instructions whenever needed.

---

# Doctor User Stories

## US-10: View Scheduled Appointments

**As a Doctor,**
I want to view my upcoming appointments,
**so that** I can prepare for patient consultations.

---

## US-11: Manage Availability

**As a Doctor,**
I want to update my available time slots,
**so that** patients can schedule appointments during available hours.

---

## US-12: Access Patient Records

**As a Doctor,**
I want to review patient information and medical history,
**so that** I can provide informed medical care.

---

## US-13: Create Prescriptions

**As a Doctor,**
I want to create prescriptions for patients,
**so that** treatment details can be recorded and shared.

---

## US-14: Update Appointment Status

**As a Doctor,**
I want to mark appointments as completed or cancelled,
**so that** appointment records remain accurate.

---

# Summary

The Smart Clinic Management System supports three primary user roles:

* Admin
* Patient
* Doctor

These user stories define the core functionality required for managing appointments, users, medical records, and prescriptions within the system.
